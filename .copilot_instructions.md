# PyQt6 & Multi-threading Strict Rules
1. **QThread Lifecycle:** NEVER allow a `QThread` object to be garbage collected or overwritten in Python while `.isRunning()` is True. Always call `.stop()` followed strictly by `.wait()` before dereferencing or calling `.deleteLater()`. Failure to do so causes `QThread: Destroyed while thread is still running` core dumps.
2. **Lazy Loading Types:** When interacting with lazy-loaded view lists (like `self.page_items` in PDFViewer), NEVER assume the type of the object. Always check `isinstance(item, QGraphicsPixmapItem)` before applying image-specific methods, as they may be placeholder `QGraphicsRectItem`s.
3. **JSON Parsing:** Do not nest JSON parsing utility calls. If `BaseAIWorker.safe_parse_json` is used, pass it raw string data, not already-parsed dictionaries or lists.
4. **PyMuPDF Memory:** Always explicitly `del` pixmaps and QImages inside background thread loops after they are converted/emitted, to prevent rapid RAM accumulation.
